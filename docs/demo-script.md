# Demo Script

## Student Flow

1. Open the student home page.
2. Confirm backend status shows connected.
3. Select “小学高年级”.
4. Review the “人工智能如何学会分类” course path.
5. Ask “机器怎么学会分类？”.
6. Confirm the answer is age-appropriate and cites the course source.
7. Run TTS loopback and use browser speech playback.
8. Open the classification animation.
9. Play, pause, and step through the animation.
10. Generate a quiz and submit an answer.
11. Confirm mastery and recommendation update.
12. Run the Python code practice panel.

## Teacher Flow

1. Open `/teacher`.
2. Confirm total learning sessions.
3. Review weak knowledge points.
4. Explain that resource upload and Dify knowledge-base sync are the next extension.

## Backup Plan

- If Dify is unavailable, local fallback answers keep the demo running.
- If Judge0 is unavailable, safe fallback explains that code is preserved and not executed locally.
- If browser speech is unavailable, text remains visible and Mock TTS loopback still validates the content.
