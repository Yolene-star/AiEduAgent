# 阶段 0 技术决策记录

本文件记录项目早期的关键工程选择。阶段 0 只做安全基线，不实现业务功能。

## 选择 Vue 作为前端方向

计划手册默认使用 Vue 3 + TypeScript。原因是 Vue 入门成本较低，组件结构清晰，适合初学团队逐步搭建学生端、教师端和演示页面。

阶段 0 不创建 `frontend/` 目录。进入阶段 2 时再初始化前端项目。

## 选择 FastAPI 作为后端方向

FastAPI 适合快速构建 JSON API，自动生成接口文档，便于用 pytest 做最小测试。它也方便后续接入大模型 API、知识库检索和学习记录接口。

阶段 0 不创建 `backend/` 目录。进入阶段 1 时只实现 `/health`。

## 选择 SQLite 作为首版本地数据存储

SQLite 不需要独立数据库服务，适合单人开发和本地演示。后续如果需要多人并发、部署或向量检索，再考虑 PostgreSQL / pgvector。

阶段 0 不创建数据库文件，所有 `*.db`、`*.sqlite`、`*.sqlite3` 都必须被忽略。

## 选择 YAML / Markdown 管理首版课程内容

U1 知识卡、题库和来源记录先使用易读、易审查的文本格式管理，方便人工核对和比赛报告引用。等内容结构稳定后，再迁移到数据库或后台管理系统。

阶段 0 只保留规划文档，不创建课程内容目录。

## 安全边界

- `.env` 只保存本地真实配置，永远不提交。
- `.env.example` 只写占位值。
- 依赖目录、缓存、数据库、日志和构建产物不进入 Git。
- 每次提交前必须查看 `git diff --cached`。
